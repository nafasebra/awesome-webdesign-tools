# Contributing

Thanks for helping grow this list! Contributions are welcome — whether you're adding a tool, fixing a typo, or improving a description.

## How to contribute

1. [Fork](https://github.com/nafasebra/awesome-webdesign-tools/fork) this repository.
2. Create a branch for your change.
3. Edit `README.md` with your addition or fix.
4. Open a [Pull Request](https://github.com/nafasebra/awesome-webdesign-tools/compare) with a clear title and short description.
5. Found a bug or broken link? [Open an issue](https://github.com/nafasebra/awesome-webdesign-tools/issues/new) instead.

## Adding a tool

Each entry follows this format:

```markdown
- [Tool Name](https://example.com) - Short description of what it does and why it's useful.
```

### Guidelines

- **Relevant** — The tool should be useful for web designers or developers (icons, colors, layouts, CSS, fonts, utilities, learning resources, AI tools, etc.).
- **Working link** — Use the tool's official URL. Avoid shorteners or affiliate links.
- **Clear description** — One sentence is enough. Focus on what the tool does, not marketing copy.
- **Right category** — Place the entry in the most fitting section. If you're unsure, mention it in your PR.
- **No duplicates** — Search the README first to make sure the tool isn't already listed.

### Categories

| Section | What belongs here |
| --- | --- |
| **Icons** | Icon libraries, SVG resources, favicon generators |
| **Colors** | Palettes, gradients, contrast checkers, color pickers |
| **Designs** | UI kits, inspiration sites, design resources |
| **Shape makers** | Clip-path, blob, pattern, and shadow generators |
| **Fonts** | Font libraries, converters, typography tools |
| **Layouts** | Grid, flexbox, and layout generators |
| **CSS loaders** | Loading spinners and animation generators |
| **Utils** | General-purpose developer and designer utilities |
| **Learning** | Tutorials, references, cheatsheets, courses |
| **Good for you** | Productivity and wellness tools for creators |
| **Components UI** | Copy-paste UI component libraries |
| **AI** | AI-powered design and development tools |

## Pull request tips

- Keep PRs focused — one tool per PR is easiest to review.
- Use a descriptive title, e.g. `Add Color Forge to Colors`.
- If you're adding your own project, that's fine — just make sure it genuinely fits the list and isn't purely promotional.

## Code of conduct

Be respectful and constructive in issues and pull requests. Spam, low-quality submissions, and unrelated self-promotion will be closed.
